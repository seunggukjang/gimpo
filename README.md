# 지역뉴스허브 — 시군구 뉴스 모니터링 + 즉시 메일 발송

GitHub Pages에 그대로 올려서 쓸 수 있는 정적 웹사이트입니다.
백엔드 서버 없이 EmailJS를 통해 **실제 이메일을 즉시 발송**합니다.

## 파일 구성
- `index.html` — 페이지 구조
- `app.js` — 뉴스 생성 로직 + EmailJS 발송 로직

## GitHub Pages 배포 방법
1. 새 GitHub 저장소 생성
2. `index.html`, `app.js` 두 파일을 루트에 업로드(또는 push)
3. 저장소 **Settings → Pages → Branch**에서 `main` (또는 `master`) 선택 후 저장
4. 몇 분 뒤 `https://<사용자명>.github.io/<저장소명>/` 으로 접속

## 메일 즉시 발송 설정 (EmailJS, 5분)

브라우저는 보안상 SMTP에 직접 연결할 수 없어서, 무료 서비스인 **EmailJS**를 거쳐 발송합니다.

1. [emailjs.com](https://www.emailjs.com) 가입 (무료 플랜: 월 200건)
2. **Email Services** → Gmail/Naver 등 본인 메일 계정 연결 → **Service ID** 복사
3. **Email Templates** → 새 템플릿 생성. 템플릿 본문에 아래 변수를 사용하세요:
   - `{{to_email}}` — 수신자
   - `{{subject}}` — 제목
   - `{{region}}` — 지역명
   - `{{news_html}}` — 뉴스 본문 (HTML)
   - `{{extra_message}}` — 추가 메시지
   - 저장 후 **Template ID** 복사
4. **Account → General** → **Public Key** 복사
5. 사이트 상단의 **"메일링 설정"** → **"발송 설정 (EmailJS)"** 탭에서 위 3개 값 입력 → 저장
6. **"연결 테스트"**로 정상 발송되는지 확인

설정값은 브라우저의 `localStorage`에 저장됩니다 (이 브라우저에서만 유지, 서버로 전송되지 않음).

## 사용 흐름
1. 지역(시·도 → 시·군·구) 선택 → 키워드 추가 → **뉴스 검색**
2. 사이드바 옆 **"메일링 설정"**에서 구독 신청 (이메일을 localStorage에 저장)
3. 상단 **"📨 지금 메일 발송"** 버튼으로:
   - 저장된 구독자 전체 또는 직접 입력한 이메일로
   - 현재 검색된 뉴스(상위 N건)를 즉시 발송
4. 개별 기사 카드의 **"✉ 이 기사 발송"**으로 특정 기사만 보낼 수도 있습니다

## 참고
- 무료 EmailJS 플랜은 월 발송량 제한(200건)이 있습니다. 대량 발송이 필요하면 유료 플랜을 검토하세요.
- 구독자 목록과 EmailJS 키는 모두 브라우저 로컬에 저장되므로, 다른 기기/브라우저에서는 별도로 설정해야 합니다.
- 뉴스 내용은 시드 기반으로 로컬에서 생성되는 데모 데이터입니다. 실제 크롤링 데이터를 쓰려면 별도 백엔드(서버리스 함수 등)가 필요합니다.
